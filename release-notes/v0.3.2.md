Alice Miner v0.3.2 — the macOS "Start does nothing" fix + auto-download + background mining.

This release fixes the v0.3.1 bug where, on macOS, pressing **Start did nothing**.

**Mac users:** if you are on v0.3.1, please download v0.3.2 (the in-app updater will also offer it now). After downloading, right-click the app → **Open** the first time (Gatekeeper).

What's fixed / new:

• **macOS Start now works.** v0.3.1 shipped a bundled mining engine whose signature drifted from its verified hash during the build, so the client refused to run it and Start silently did nothing. The build no longer re-signs bundled engines, and a release gate now re-checks every packaged engine's hash — so the engine always matches what the client expects.

• **Errors are now visible.** If mining can't start for any reason, the app shows a clear message instead of doing nothing.

• **Auto-update is reachable again.** v0.3.1 couldn't be reached by the in-app updater from v0.3.0; that's fixed, and the app now checks for updates at launch and shows a dot on Settings when one is available.

• **Engines self-install.** If a required mining engine is missing, the app downloads it automatically and verifies it against a built-in hash before running it — nothing unverified is ever executed.

• **Background mining (macOS).** New "Keep mining when closed" toggle in Settings: mining keeps running after you close the window and restarts at login. Your reward address stays in your keystore and is never written into the background service. (CPU lane; also available headless via `alice-miner-cli service --install`.)

• **Multi-GPU.** `alice-miner-cli detect` now lists each GPU with its index so you can target specific cards with `--gpus`.

Same mining lanes, proof-of-possession, credit-only rewards, and ed25519-signed auto-update as v0.3.0/v0.3.1.
